# Architecture Decision Record: Hello User API & UI

**Date:** 2026-03-17
**Role:** `@techlead`
**Status:** Accepted

## 1. Context
We need to establish a basic end-to-end communication channel between our frontend and backend using a simple `POST` endpoint that accepts a user's name and returns a personalized greeting string.

## 2. API Contract (Backend)

We will expose a single REST API endpoint.

**Endpoint:** `POST /api/hello`
**Content-Type:** `application/json`

### Request Schema (Input Validation)
The backend must validate this structure (e.g., using `Zod`).
```json
{
  "name": "string (required, min length 1)"
}
```

### Response Schema (Success)
**Status:** `200 OK`
```json
{
  "success": true,
  "message": "Xin chào thân ái, [Tên]!"
}
```

### Response Schema (Error - Validation Failed)
**Status:** `400 Bad Request`
```json
{
  "success": false,
  "message": "Invalid input: Name is required."
}
```

## 3. Project Structure & Standards

### Frontend (React/Next.js)
Since this is a simple UI feature, we will place the logic inside the feature-based folder structure:
- `components/hello-user/HelloUserForm.tsx` (Handles UI and state)
- `services/api.ts` (Handles the `fetch` or `axios` POST call to `/api/hello`)

**Styling:** Tailwind CSS (utility-first) or module CSS.
**State:** `useState` is sufficient for the input and result. Global state (Zustand/Redux) is **not** required.

### Backend (Node.js/Express)
Following our architectural layer guidelines:
- **Routes:** `routes/hello.ts` -> Maps `POST /api/hello` to the controller.
- **Controllers:** `controllers/HelloController.ts` -> Parses request, calls service, returns JSON.
- **Services:** `services/HelloService.ts` -> Contains the business logic (string interpolation `"Xin chào thân ái, " + name + "!"`).

**Database:** This feature requires no database. The Repository layer will be bypassed for this specific route.

## 4. Consequences & Risks
- **CORS:** Ensure the backend configures Cross-Origin Resource Sharing (CORS) correctly so the frontend can successfully POST to `/api/hello` during local development.
- **Security:** Ensure the backend does basic XSS sanitization if necessary, although since we are just echoing it back safely in JSON, standard JSON parsing libraries cover most of the risk.
