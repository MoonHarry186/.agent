## 1. Context
We need to implement a simple "Hello User" feature that ensures end-to-end communication works smoothly, strictly adhering to our new Backend and Frontend Developer rules including separate `server/` and `client/` directories, Git branching requirements, and centralized routing.

## 2. API Contract (Backend)

We will expose a single REST API endpoint.

**Endpoint:** `POST /api/hello`
**Content-Type:** `application/json`

### Request Schema (Input Validation)
The backend must validate this structure (e.g., using `Zod` or `Joi`).
```json
{
  "name": "string (required, min length 1)"
}
```

### Response Schema (Success)
**Status:** `200 OK`
```json
{
  "success": true,
  "message": "Xin chào thân ái, [Tên]!",
  "data": null
}
```

### Response Schema (Error - Validation Failed)
**Status:** `400 Bad Request`
```json
{
  "success": false,
  "message": "Validation Error: Name is required."
}
```

## 3. Architecture & Structural Enforcement

### Backend (Node.js/Express)
1. **Branching:** Developers MUST run `git checkout -b feature/hello-user-backend` before writing code.
2. **Directory:** All code MUST reside in `worksflow/server/`.
3. **Central Route:** All routes must aggregate through a central route file (e.g., `server/src/routes/index.ts` connecting to `server/src/app.ts`).
4. **Structure:** Follow feature-based modules:
   - `server/src/features/hello/hello.controller.ts`
   - `server/src/features/hello/hello.service.ts`
   - `server/src/features/hello/hello.routes.ts`

### Frontend (React/Next.js/Vite)
1. **Branching:** Developers MUST run `git checkout -b feature/hello-user-frontend` before writing code.
2. **Directory:** All code MUST reside in `worksflow/client/`.
3. **Structure:** Use functional components and separate API fetching logic (e.g., `client/src/services/api.ts`).
4. **UI Validation:** Ensure "name" input is not empty before submitting the fetch request. Handle potential network/CORS errors gracefully without breaking the layout.

## 4. Known Risks
- **CORS Configuration:** Ensure `server/src/app.ts` implements the `cors` middleware, or the frontend won't be able to communicate during local testing across different ports.
