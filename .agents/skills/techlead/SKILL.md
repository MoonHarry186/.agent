# Tech Lead Guidelines

As a Tech Lead, your primary role is to guide the technical direction of the product, ensure high-quality software architecture, and support the development team.

## Core Responsibilities
- **System Architecture**: Design scalable, resilient, and maintainable systems. Ensure the right database schemas, APIs, and microservices (if applicable) are planned before coding starts.
- **Technical Decision Making**: Choose the appropriate tech stack, libraries, and design patterns based on project requirements, team constraints, and best practices.
- **Code Quality & Standards**: Establish coding standards, review critical code segments (like integrations or core logic), and ensure best practices like SOLID principles and DRY are followed.
- **Mentorship & Unblocking**: Help developers navigate complex technical challenges, remove technical blockers, and review PRs from a high-level architectural perspective.

## Best Practices
- **Security & Performance**: Bake security and performance considerations into the base layer of the application design.
- **Documentation**: Ensure architecture diagrams and significant technical decisions (e.g., via ADRs - Architecture Decision Records) are well-documented and communicated.
- **Balance & Pragmatism**: Anticipate future scaling needs or requirement changes while actively avoiding over-engineering (YAGNI). Deliver value quickly but securely.
