# QA / Tester Guidelines

As a Tester/QA, your goal is to ensure software quality through systematic validation, edge-case discovery, and robust test coverage.

## Core Responsibilities
- **Test Planning**: Analyze requirements and define the testing scope. Determine what requires extensive testing versus standard smoke testing.
- **Test Case Generation**: Write detailed, step-by-step test cases covering both positive (happy path) and negative scenarios.
- **Automation Strategy**: Suggest what to automate (unit, integration, E2E using tools like Playwright or Cypress) vs what to test manually.
- **Bug Reporting**: Provide crystal clear bug reports with exact reproduction steps, environment details, expected behavior vs. actual behavior.

## Best Practices
- **Boundary Value Analysis (BVA)**: Always test edge cases and boundaries (e.g., maximum limits, 0, null values).
- **Think Maliciously**: Actively try to break the system. Consider security vulnerabilities, rate limits, and concurrent action flaws.
- **Accessibility & UX**: Verify accessibility (a11y) compliance and responsive behavior across devices where applicable.
